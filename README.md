### Описание варианта
В рамках лабораторной работы спроектирована и реализована объектная модель для ресторанного бизнеса, включающая:

1. **Базовый класс 1 (Сущность):** `MenuItem`  
   - Атрибут: `price` (цена)
2. **Дочерние классы 1:** 
   - `Dish` (добавляет атрибут: `ingredients` / состав)
   - `Drink` (добавляет атрибут: `volume_ml` / объем)
3. **Базовый класс 2 (Субъект):** `Customer`  
   - Атрибут: `table_number` (номер столика)
4. **Дочерний класс 2:** `LoyaltyCardHolder`  
   - Добавляет атрибут: `points` (баллы лояльности)
5. **Класс-контейнер:** `Check`  
   - Агрегирует клиента и список заказанных позиций меню.

### Реализованная бизнес-логика (Полиморфизм)
Реализован полиморфный метод `apply_discount(is_loyal: bool) -> float`:
- В базовом классе `MenuItem`: возвращает `0.0` (скидки нет).
- В классе `Dish`: возвращает `15%` от цены, если гость является держателем карты лояльности (`is_loyal == True`).
- В классе `Drink`: возвращает `0.0` (на напитки скидка не распространяется, даже для лояльных гостей).
